## 2. 自动绑定判定逻辑

### 2.1 功能逻辑判定

#### 2.1.1 自动绑定货权
- 启用
  - 仓库维度参与业务逻辑判定
- 禁用
  - 仓库维度不参与业务逻辑判定
  - 即使满足业务条件也不在自动绑定任务执行范围

#### 2.1.2 允许吨位差
- 启用
  - 自动绑定执行的过户单容错率为：
  - （未关联重量 / 总收货重量）小于或等于 0.3%
- 禁用
  - 自动绑定执行的过户单数量重量必须等于匹配到的收货单数量重量

### 2.2 业务逻辑判定

#### 2.2.1 找钢网-预售预购租户（配置化）
- 不参与货权自动绑定

#### 2.2.2 非 找钢网-预售预购租户

##### 2.2.2.1 没有开启货权自动绑定
- 不参与货权自动绑定
  - 执行收货单明细定时任务
    - 收货单完结（参考之前的完结功能逻辑）

##### 2.2.2.2 开启货权自动绑定
- 售后状态 = 0 实提，出库后
  - 不参与自动绑定，手动处理
- 售后状态 != 0 实提，出库后
  - 过户单订单标签 = 预售订单、运营敏采转销售、运营敏采（系统）
    - 不参与自动绑定，手动处理
  - 过户单订单标签 != 预售订单、运营敏采转销售、运营敏采（系统）
    - 当前胖猫云无此标签，后面一朵云融合之后 PMY 也会加上此标签
    - 不满足：仓储确认状态 = 待确认 && 关联收货单状态 = 未关联、部分关联 && 过户状态 = 过户完成 && 售后状态 = 其它
      - 不参与自动绑定，手动处理
    - 满足：仓储确认状态 = 待确认 && 关联收货单状态 = 未关联、部分关联 && 过户状态 = 过户完成 && 售后状态 = 其它
      - 过户单明细已关联吨位/数量 != 0
        - 不参与自动绑定，手动处理
      - 过户单明细已关联吨位 = 0，数量 = 0
        - 不满足：收货单状态 = 批准未完结 && 关联状态 = 未关联、部分关联 && 收货单类型 ≠ 库内转入 && 外部映射状态 = 成功
          - 不参与自动绑定，手动处理
        - 满足：收货单状态 = 批准未完结 && 关联状态 = 未关联、部分关联 && 收货单类型 ≠ 库内转入 && 外部映射状态 = 成功
          - 不满足：过户单进入云仓日期 - 收货单收货日期 | <= 1 天（按天计算，去掉时分秒，取绝对值）
            - 不参与自动绑定，手动处理
          - 满足：过户单进入云仓日期 - 收货单收货日期 | <= 1 天（按天计算，去掉时分秒，取绝对值）
            - 过户单供应商 != 收货单原货主（去掉前后空格和中间空格，剔除“公司”“有限公司”等文字之后精准匹配）
              - 不参与自动绑定，手动处理
            - 过户单供应商 = 收货单原货主（去掉前后空格和中间空格，剔除“公司”“有限公司”等文字之后精准匹配）
              - 过户单明细与收货单明细 SKU 匹配（捆包号校验）
                - 不匹配
                  - 不参与自动绑定，手动处理
                - 匹配
                  - 允许吨位差前提是数量一致（收货单明细必须纯洁无业务操作）
                    - 启用
                      - 满足：过户单明细预提件数 = 收货单明细收货件数 && |过户单明细预提吨位 - 收货单明细收货吨位| <= 吨位差值 && 收货单明细已关联吨位 = 0 && 收货单明细已关联件数 = 0
                        - 过户单明细与收货单明细货权自动绑定并记录操作日志
                      - 不满足：过户单明细预提件数 = 收货单明细收货件数 && |过户单明细预提吨位 - 收货单明细收货吨位| <= 吨位差值 && 收货单明细已关联吨位 = 0 && 收货单明细已关联件数 = 0
                        - 不参与自动绑定，手动处理
                    - 禁用
                      - 满足：过户单明细预提件数 = 收货单明细收货件数 && |过户单明细预提吨位 - 收货单明细收货吨位| <= 吨位差值 && 收货单明细已关联吨位 = 0 && 收货单明细已关联件数 = 0
                        - 过户单明细与收货单明细货权自动绑定并记录操作日志
                      - 不满足：过户单明细预提件数 = 收货单明细收货件数 && |过户单明细预提吨位 - 收货单明细收货吨位| <= 吨位差值 && 收货单明细已关联吨位 = 0 && 收货单明细已关联件数 = 0
                        - 不参与自动绑定，手动处理
                  - 品线判定
                    - 热卷、冷镀
                      - 去掉空格精准校验捆包号
                      - 以下划线截取之后精准匹配，例如：收货单 `123_456`，过户单 `123` 可以，但 `12` 不行
                    - 非热卷、冷镀
                      - 无需检验捆包号

## 3. 仓储确认定时任务
- 过户单数据满足：
  - 仓储确认状态 = 待确认
  - 过户单明细状态 = 部分关联
  - 过户单明细关联件数 = 预提件数
  - （预提吨位 - 已关联吨位）/ 预提吨位 <= 0.003
  - 满足部分条件
    - 执行定时任务后，仓储确认状态保持不变
  - 满足所有条件
    - 执行定时任务后，仓储确认状态 = 已确认
    - 过户单记录操作日志
- 过户单数据不满足上述条件
  - 执行定时任务后，仓储确认状态保持不变

## 4. 收货单明细定时任务
- 满足：
  - 收货单状态 = 部分关联
  - 收货单明细未关联件数 = 0
  - （未关联吨位 / 收货吨位）<= 0.003
  - 满足部分条件
    - 执行定时任务后保持不变
  - 满足所有条件
    - 收货单关联状态保持不变
    - 收货单明细完结状态 = 已完结
    - 收货单记录操作日志
      - 操作人：系统
      - 操作时间：收货单完结时间
      - 操作类型：收货单完结
      - 备注：系统自动绑定货权
- 不满足上述条件
  - 执行定时任务后保持不变

## 5. 历史数据初始化以及检查
- 自动绑定货权
  - 历史的没开变为开之后，任务执行是否正常
- 允许吨位差
  - 历史的不允许改为允许后，任务执行是否正常

## 6. 手动相关功能回归

### 6.1 手动货权绑定
- 查询关联
  - 过户单关联量检查
  - 收货单关联量检查

### 6.2 手动仓储确认
- 手动仓储确认
  - 过户单以及上游仓储确认状态检查
- 手动仓储驳回
  - 过户单以及上游仓储确认状态检查

### 6.3 解绑
- 货权解绑状态以及解绑后关联量检查
- 明细维度如果操作过解绑，就不再参与自动绑定执行任务

### 6.4 手动完结
- 收货单完结执行后状态检查
  - 关联查询无该数据
  - 收货单明细完结状态同步
